# 香山验证知识库

基于 XiangShan RISC-V 处理器的 spec 文件和代码分析，总结验证关键信息。

## 1. SpecBlock 模块分析

### 模块功能
`SpecBlock_v4` 是一个规格化块（Speculation Block），用于处理除法运算的规格化逻辑。

### 关键信号
- `io_q_j[4:0]`: 商的选择信号（5位）
- `io_cons_*`: 约束条件数组（多个9位输入）
- 输出：规格化后的结果

### 验证要点
1. **边界值测试**：测试 `io_q_j` 的所有可能值（0-31）
2. **约束条件组合**：测试不同 `io_cons` 组合
3. **数据路径覆盖**：确保所有规格化路径都被执行

## 2. 香山处理器关键模块

### L2 Cache 相关模块
- **L2Cache**: L2 缓存主控制器
- **L2DataStorage**: L2 数据存储
- **L2Directory**: L2 目录管理
- **L2TLB**: L2 转换后备缓冲区
- **L2TLBWrapper**: L2 TLB 包装器
- **L2TlbPrefetch**: L2 TLB 预取
- **L2Top**: L2 顶层模块

### 验证策略
1. **缓存一致性**：测试多核场景下的缓存一致性
2. **TLB 管理**：测试页表转换和 TLB 替换策略
3. **预取机制**：测试数据预取的有效性
