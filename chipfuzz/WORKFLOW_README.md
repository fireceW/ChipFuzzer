# ChipFuzzer 验证流程图使用说明

## 功能概述

验证流程图展示了 ChipFuzzer 框架的完整验证流程，从 LLM 生成测试用例到编译、执行仿真、收集覆盖率，再到判断结果和保存成功用例的完整闭环。

## 流程图步骤

1. **分析未覆盖代码** 🔍
   - 提取目标模块的未覆盖代码行
   - 分析代码结构和功能

2. **LLM 生成测试用例** 🤖
   - 基于未覆盖代码生成 RISC-V 汇编代码
   - 使用大语言模型智能生成测试用例

3. **编译汇编代码** ⚙️
   - 将 .S 文件编译为 .elf 文件
   - 转换为 .bin 二进制文件

4. **执行仿真** 🚀
   - 在香山模拟器中运行测试用例
   - 收集执行过程中的覆盖率数据

5. **收集覆盖率** 📊
   - 提取代码覆盖率数据
   - 合并到全局覆盖率统计

6. **判断结果** ✅
   - 分析覆盖率是否提升
   - 判断测试用例是否成功

7. **保存成功用例** 💾
   - 将成功用例保存到种子库
   - 用于后续测试用例生成

8. **继续迭代** 🔄
   - 返回步骤 1 继续生成
   - 形成自动化验证闭环

## 使用方法

### 1. 查看流程图

流程图会自动显示在网页的"验证流程"卡片中。页面加载时会自动播放演示动画。

### 2. 实时状态显示

当验证任务运行时，流程图会根据日志内容自动高亮当前执行的步骤：
- 当前步骤会显示高亮效果
- 连接线会显示流动动画
- 步骤节点会有发光效果

### 3. 重置动画

点击"重置动画"按钮可以重新播放演示动画，展示完整的流程。

### 4. 导出图片（用于PPT）

点击"导出图片"按钮可以将流程图导出为 PNG 图片：
- 图片为高分辨率（2倍缩放）
- 适合插入到 PPT 中
- 文件名格式：`ChipFuzzer_Workflow_时间戳.png`

## 技术实现

### 文件结构

- `chipfuzz/assets/workflow.js` - 流程图核心逻辑
- `chipfuzz/assets/style.css` - 流程图样式
- `chipfuzz/index.html` - 流程图容器

### 主要特性

1. **SVG 矢量图形**
   - 使用 SVG 绘制，支持任意缩放
   - 清晰的视觉效果，适合展示

2. **实时状态更新**
   - 监听日志输出，自动识别当前步骤
   - 动态高亮显示

3. **动画效果**
   - 平滑的过渡动画
   - 流动的连接线效果
   - 节点悬停效果

4. **响应式设计**
   - 适配不同屏幕尺寸
   - 移动端友好

## 流程图布局

```
[分析] → [生成] → [编译] → [仿真] → [覆盖率]
                                    ↓
[迭代] ← [保存] ← [判断] ←─────────┘
```

## 自定义配置

如需修改流程图，可以编辑 `workflow.js` 文件：

- `workflowSteps` - 定义流程步骤
- `connections` - 定义步骤之间的连接
- `createStepNode()` - 自定义节点样式
- `createConnectionPath()` - 自定义连接线样式

## 注意事项

1. 流程图需要页面加载完成后才能显示
2. 导出功能需要浏览器支持 Canvas API
3. 实时状态更新依赖于日志输出格式
4. 建议使用现代浏览器（Chrome、Firefox、Edge）

## 常见问题

**Q: 流程图不显示？**
A: 检查浏览器控制台是否有错误，确保 `workflow.js` 已正确加载。

**Q: 导出图片失败？**
A: 可能是浏览器安全限制，尝试使用浏览器的截图功能作为备选方案。

**Q: 如何修改流程步骤？**
A: 编辑 `workflow.js` 中的 `workflowSteps` 数组，添加或修改步骤定义。

**Q: 流程图可以用于商业展示吗？**
A: 可以，流程图设计用于展示和PPT演示，完全可以使用。
