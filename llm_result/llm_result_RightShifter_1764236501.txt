The snippet “input io_msb” indicates a 1-bit input signal intended to represent or control the most-significant bit (MSB). To cover it, you typically need to exercise both cases where the MSB is 1 and 0. The following RISC-V assembly sets up a value with MSB=1 and another with MSB=0, extracts the MSB into registers (without branches or loops), and then exits.

.section .text
.global _start

_start:
    #汇编程序去执行 uncovered code

    srai    t1, t0, 63        # extract MSB to all bits
