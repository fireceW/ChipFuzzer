下面是符合你模板要求、可以直接汇编的 RISC‐V 程序:

```assembly
.section .text
.global _start

_start:
    # 下面这几条只是占位计算,避免空程序
    # 在你的 testbench 中,可在 CPU 执行到这些指令时,配合外部逻辑
    # 去拉高 start_handshaked、设置 fsm_q 等,以覆盖 Verilog 代码。

    li      a0, 0x4      # 模拟某种状态值
    li      a1, 0x8      # 模拟另一种状态值
    li      a2, 0x1      # 可在 TB 中用作“start_handshaked=1”的触发参考
    li      a3, 0xC      # 用作迭代次数等的占位
    li      a4, 0x5
    li      a5, 0xF
    li      a6, 0xA
    li      a7, 0x3      # 注意:a7 也用作 ecall 编号寄存器,后面会重新赋值

    # 稍微做一些组合运算,占位以便在波形里观察寄存器变化
    add     a0, a0, a1   # a0 = 0x4 + 0x8 = 0xC
    xor     a1, a2, a3   # a1 = 0x1 ^ 0xC
    and     a2, a4, a5   # a2 = 0x5 & 0xF
    or      a3, a6, a7   # a3 = 0xA | 0x3

    # 下面为汇编程序退出逻辑,请不要修改
    li      gp, 1
    li      a7, 93       # Linux RISC-V exit syscall
    li      a0, 0        # exit code 0
    ecall
    unimp
    .insn   2, 0x0000
```

1. 没有使用 `t7`、`t8` 等寄存器别名,避免 “illegal operands `li t7,0x1`” 这类错误。  
2. 代码是严格的顺序执行,无循环、无分支跳转。
